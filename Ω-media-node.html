مصادر الترافيك بقت متشظية (ميتا، جوجل، QR، واتساب، إحالات). كل مصدر بيحمل نية مختلفة. الصفحة الثابتة بتفرض سرد واحد… فتقلل جودة التحويل وتضعف هيبة البراند.

Static pages became an insult to modern attention. The void does not accept “one view for all”. We shape experiences based on context—without begging the user for inputs.

05) Mode Engine (Deterministic) ٥) محرك المودات (حاسم) 05) Mode Matrix (Void Rules) ENGINE

Modes are not “themes”. Modes are behavior contracts. They decide: ordering, messaging, density, motion, and CTA.

المودات مش “ثيمات”. المودات هي عقود سلوك. بتحدد الترتيب، الرسائل، الكثافة، الحركة، والـ CTA.

A mode is a stance. The node selects a stance based on context. This prevents random UI and kills drift.

  • SILENT → minimal, calm, observe-first.
  • CONTACT → action-first, WhatsApp/Call/Email.
  • BUSINESS → proof-first, cases, portfolio.
  • FOUNDER → signature, authority, cinematic.
  • SILENT → بسيط وهادئ (مراقبة أولًا).
  • CONTACT → تنفيذ سريع (واتساب/اتصال/إيميل).
  • BUSINESS → إثبات وقصص نجاح.
  • FOUNDER → توقيع وهيبة وسينماتيك.
  • Modes are the discipline of the surface.

Example Determinism مثال على الحسم Rule Sketch NO DRIFT

  • IF source == "qr" → CONTACT
  • IF source == "linkedin" → BUSINESS
  • IF source == "instagram" → SILENT
  • IF internal token → ATLAS/FOUNDER
  • لو المصدر QR → CONTACT
  • لو لينكدإن → BUSINESS
  • لو إنستجرام → SILENT
  • لو داخلي → ATLAS/FOUNDER
  • Rules are explicit. Silence is never used.
