01) Purpose

Ω Media Link is not a “linktree”. It is a public-facing Omega Node that acts as an identity gateway. This reference defines the awareness model, mode engine, data contract, and rendering rules.

ScopePublic Gateway / Identity Surface
CoreContext + Intent + Mode Engine
DataSingle Source of Truth (contract)
RiskDrift if identity is ambiguous

02) Definition WHAT IT IS

Ω Media Link is a surface identity node that:

  • Detects context (UTM / referrer / device / time).
  • Infers intent (contact / work / observe / explore).
  • Selects a mode (silent / business / contact / founder / internal).
  • Renders UI accordingly (links, order, copy, tone, animations).

It must remain minimal, calm, and authoritative. It never “explains”; it routes.

03) Awareness Model WHAT “AWARE” MEANS

  • Self-Awareness: identity, owner, role, tone.
  • Context-Awareness: where/from/when/how opened.
  • Visitor-Awareness: language/device/returning.
  • Intent-Awareness: inferred purpose.
  • Mode Selection: deterministic mode engine.
  • Signals: optional event emission (memory-friendly).

04) Architecture LAYERS

Ω-Media-Link
├─ Identity Layer     (Who am I?)
├─ Context Layer      (Where/when/from?)
├─ Intent Layer       (Why opened?)
├─ Mode Engine        (Which persona?)
├─ Renderer           (UI as a result)
└─ Signal Layer       (Optional)

The renderer must not decide behavior. All decisions come from Mode Engine using the contract.

06) Mode Matrix DETERMINISTIC

Example mapping rules (simplified):

// Inputs: source, referrer, utm, device, language, visit
// Output: mode

IF context.source == "qr"            → CONTACT
IF context.source == "instagram"      → SILENT
IF context.source == "linkedin"       → BUSINESS
IF context.trust == "internal"        → INTERNAL (locked)
IF context.ownerView == true          → FOUNDER
ELSE                                  DEFAULT (SILENT)

Modes define: link ordering, copy tone, density, motion level, and CTA type.

07) Rendering Rules UI IS A RESULT

  • Never show “everything” to everyone.
  • Sort links by mode priority.
  • Copy must be short and directive (no long explanations).
  • Motion level depends on mode (cinema only in FOUNDER view).
  • Accessibility: reduced motion support is mandatory.

08) Signals / Memory OPTIONAL

Signals are not “tracking”. They are minimal event emissions for Omega awareness. 

{
  "node": "Ω-MEDIA",
  "event": "MODE_ENTERED",
  "mode": "CONTACT",
  "timestamp": "ISO8601",
  "confidence": 0.82
}

If implemented: emit-only by default. No server dependency required.

09) Implementation Checklist PHASES

  • Phase 1: identity contract + context detection + intent inference
  • Phase 2: mode engine + renderer mapping + UI toggles
  • Phase 3: signals (optional) + export + audit logs
  • Phase 4: internal locked mode (optional) + permissions

“No mode without contract.” “No UI decision inside components.”

11) Governance ANTI-DRIFT

  • Identity owner and canonical roles must never be ambiguous.
  • Any rename requires contract update + changelog entry.
  • Internal modes must be locked (no accidental exposure).
  • Do not rely on silent assumptions; rules must be explicit.
